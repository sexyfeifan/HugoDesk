# 更新说明（Changelog）

## v0.3.11 - 2026-03-04

### 版本目标 🎯

解决发布页与项目页配置重复、补齐 Hugo 运维快捷操作，并让预览更接近最终发布效果。

### 主要调整 ✅

- 远程地址与凭据统一在“项目”页配置，发布页改为读取型状态展示。
- 新增重复 Pages workflow 检测与自动清理，减少同一次提交触发重复 Deploy 的问题。
- 新增 Hugo 快捷操作：
  - 检查 Hugo 版本
  - 升级 Hugo（Homebrew）
- 编辑器预览改为 Hugo 最终渲染预览（构建后从 `public` 读取 HTML）。
- 新增“刷新最终预览（构建）”按钮，提升预览可控性。

## v0.3.10 - 2026-03-04

### 版本目标 🎯

解决配置重复、补齐 Hugo 常用快捷操作，并将编辑预览改为更接近最终发布结果的渲染方式。

### 主要调整 ✅

- 远程地址与凭据配置统一到“项目”页，去除发布页重复输入项。
- 发布页 Actions 状态改为读取型展示，避免双处配置不一致。
- 新增 Hugo 工具快捷操作：
  - 检查 Hugo 版本
  - 升级 Hugo（Homebrew）
- 编辑器预览改为 Hugo 最终渲染预览：
  - 切换到预览时自动构建
  - 支持“刷新最终预览（构建）”按钮
  - 从 `public` 目录读取对应 HTML 文件显示

### 结果

- 发布流程更聚焦，配置入口更清晰。
- 常见维护命令可在 UI 内直接执行。
- 预览结果与最终发布效果一致性更高。

## v0.3.9 - 2026-03-04

### 版本目标 🎯

本版本聚焦于“减少过度设置，固定 Hugo 正确发布路径”，让 HugoDesk 更稳定地替代命令行发布博客。

### 主要调整 ✅

- 固定发布策略为 GitHub Actions（移除易误导的发布分支策略切换）。
- 移除“推送时排除 `hugo.toml`”选项，确保 Hugo 关键配置始终参与发布。
- 精简发布控制台功能，只保留核心流程入口。
- 精简项目页快捷操作与项目配置包操作，减少重复按钮。
- 保留并强化“一键生成 Pages Workflow”能力，用于补齐 GitHub Pages 部署链路。
- 清理无入口旧逻辑，降低后续维护复杂度。

### 发布流程变化 📦

发布页主流程调整为：

1. 同步远程
2. 一键检测推送与部署
3. 提交并推送

### 兼容与安全 🔒

- `.hugodesk.local.json` 仍作为项目本地配置包使用，不进入 Git。
- Token 仍通过安全机制参与发布，不在日志中明文泄露。

### 历史版本

- v0.3.8：引入 Pages Workflow 一键生成与部署链路检测。
- v0.3.7：修复目录定位、编辑器工具和预览图片路径等关键问题。
