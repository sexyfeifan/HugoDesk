# 更新说明（Changelog）

## v0.3.25 - 2026-03-05

### 版本目标 🎯

修复发布执行区 Workflow 时间显示与本地时区不一致的问题，并完成新版本打包归档。

### 主要调整 ✅

- 发布页 `Workflow` 信息中的 `创建时间/更新时间`：
  - 不再直接显示 GitHub API 返回的 UTC 字符串（`...Z`）。
  - 统一转换为系统本地时区时间显示（格式：`yyyy-MM-dd HH:mm:ss zzz`）。
- 字段标题同步更新为：
  - `创建时间（本地）`
  - `更新时间（本地）`

## v0.3.24 - 2026-03-05

### 版本目标 🎯

稳定发布页预检卡片排布，并升级主题设置为“可检测、可切换、可按主题能力精简”。

### 主要调整 ✅

- 发布前检查区改为固定高度卡片：
  - 卡片内不再直出长错误文本，避免信息变化导致卡片高度波动和排布不齐。
  - 点击卡片后弹出详情窗口查看完整信息。
- 发布前检查顺序微调：
  - `GitHub 连通性` 移至预检区域末尾显示。
- 主题设置新增“主题检测与选择”卡片：
  - 自动扫描 `themes/` 目录检测可用主题。
  - 当前 `theme` 不在本地目录时，仍保留为可选项（兼容 Hugo Modules）。
  - 支持手动输入主题名并应用。
- 主题能力感知设置：
  - 根据主题文件中检测到的能力标签，动态精简设置项。
  - 对 `Gitalk`、`搜索输出`、`外链`、`数学公式` 相关设置做按主题显示/隐藏。

## v0.3.23 - 2026-03-05

### 版本目标 🎯

解决发布页 Workflow 状态“显示旧数据”的问题，让状态自动同步最新运行记录。

### 主要调整 ✅

- GitHub Actions 状态查询改为更精准策略：
  - 优先按 `hugo.yaml` workflow 文件查询；
  - 同时按发布分支过滤（如 `main`），减少拿到历史/无关 workflow 运行记录的概率。
- 增加 Workflow 状态自动同步：
  - 应用启动后自动拉取一次最新运行状态；
  - 每 30 秒后台自动刷新一次。
- 发布页新增“最近同步时间”显示，便于确认当前信息是否为最新。

## v0.3.22 - 2026-03-05

### 版本目标 🎯

增强软件内预览可用性与发布日志可读性，并补齐自动化发布前检查能力。

### 主要调整 ✅

- 写作页预览增强：
  - 增加“弹窗预览”，可在软件内独立窗口查看预览。

  - 调整为 Markdown 语法体检（符号正确性、结构修正、清理无意义字符）。
  - 新增进度显示（状态文本 + 进度条）。
- 发布页日志区优化：
  - 日志行宽加宽并左对齐。
  - 增加行号列与颜色标注（时间行/章节行/错误行）。
  - 新增“刷新日志”按钮。
- 发布前检查增强：
  - 新增“GitHub 连通性”检查（显示 ping RTT）。
  - 连通性每 30 秒自动刷新一次。
  - 应用启动后自动执行一轮完整发布前检查（含 Pages 来源与连通性）。

## v0.3.20 - 2026-03-05

### 版本目标 🎯

修复写作页点击“预览”触发的稳定复现闪退。

### 主要调整 ✅

- 重构实时预览调度机制：
  - 移除 `DispatchWorkItem` 直接在后台执行的实现。
  - 改为主线程 `Task` 调度 + `nonisolated` 后台构建函数。

- 解决 `AppViewModel.scheduleLivePreviewRefresh` 在后台线程触发 `@MainActor` 隔离断言导致的 `SIGTRAP` 崩溃。

## v0.3.19 - 2026-03-05

### 版本目标 🎯

修复 GitHub 仓库名包含 `github` 时 Pages/Actions 仓库解析错误，避免生成错误设置链接。

### 主要调整 ✅

- 修复仓库名解析逻辑：
  - 旧逻辑会把仓库名中的 `.git` 子串全部替换，导致 `sexyfeifan.github.io.git` 被误解析为 `sexyfeifanhub.io`。
  - 新逻辑仅移除末尾 `.git` 后缀，保留仓库名中合法的 `github` 字符串。
- 同步修复 `GitHubPagesService` 与 `GitHubActionsService` 两处解析实现，保证 Pages 设置链接与 Actions 查询仓库一致。

## v0.3.18 - 2026-03-04

### 版本目标 🎯

修复写作与发布页关键交互问题，统一发布逻辑，并增强 GitHub Token 兼容性。

### 主要调整 ✅

- 修复写作页进入后顶部选项栏漂移：移除会触发布局抖动的页面工具栏入口。
- 写作页“显示预览区”改为 `预览`，并优化实时预览刷新调度，修复点击预览崩溃问题。
- 发布前检查固定为以下顺序：
  - 项目配置文件
  - 本地配置包
  - 推送地址
  - 部署策略
  - GitHub Token
  - Pages 来源
  - Pages workflow
  - 重置 Pages workflow
  - 文章数量
  - 当前主题
  - Git 冲突
- 发布错误信息改为全局弹窗提示，避免打乱预检区排版。
- 远程凭据支持双 Token：
  - Fine-grained PAT 输入框
  - Classic PAT 输入框
  - API 检测优先使用 Classic，兼容 Pages 来源检测权限差异。
- `一键发布` 与 `提交并推送` 统一为同一发布管线（保留两个按钮，行为一致）。
- 发布提交信息默认值设为 `通过 HugoDesk 发布博客更新  🎉`，并可在发布页直接编辑。

## v0.3.16 - 2026-03-04

### 版本目标 🎯

修复发布主流程中影响可用性的关键问题，并提升对 Hugo 官方目录/配置实践的兼容性。

### 主要调整 ✅

- 修复 `Hugo 文件结构修复` 无法真正创建缺失配置文件的问题。
- 一键发布流程补齐与手动发布一致的前置步骤：
  - 自动保存当前编辑中的文章
  - 自动归一化图片链接
- 生成的 Pages workflow 增加 `submodules: recursive`，兼容主题使用 git submodule 的仓库。
- 文章扫描改为递归遍历 `content` 子目录，支持更常见的 Hugo 内容结构。
- 配置文件识别与预检提示统一支持多种官方配置路径（`hugo.* / config.* / config/_default/*`）。
- 优化主题配置保存策略：改为“增量更新已知键”，避免覆盖未知 TOML 字段与注释。
- 发布页“打开运行详情”移除强制解包，避免异常 URL 导致崩溃。
- 预览读权限扩大到 `public` 根目录，减少资源引用失败。
- 命令执行输出改为临时文件落地读取，降低大输出时流程卡住风险。
- 重复 workflow 清理规则收紧，降低误删其他 CI workflow 的风险。

## v0.3.15 - 2026-03-04

### 版本目标 🎯

跑通并简化“创建文章 → 推送 GitHub → 触发 Action → 生成网页”主流程。

### 主要调整 ✅

- 修复一键发布被“推荐目录缺失”错误阻断的问题（仅必需项缺失才阻断）。
- Hugo 结构检测支持多种官方配置路径：
  - `hugo.toml / hugo.yaml / hugo.yml / hugo.json`
  - `config.*`
  - `config/_default/hugo.*` 与 `config/_default/config.*`
- 一键发布流程简化为：
  - 结构检查（仅必需项）
  - 同步远端
  - 自动补齐 workflow
  - Pages 来源检查/修复（失败仅告警）
  - 提交并推送
  - 查询最新 Action 状态
- 发布页操作重排：默认推荐“一键发布”，手动模式简化为“同步远端 → 提交并推送 → 部署状态”。
